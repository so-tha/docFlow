Informações de funcionamento do sistema.

Ciclo de Vida do Documento

O DocFlow é um sistema de controle de versão projetado para planilhas e relatórios estruturados. O ciclo de vida de um documento segue estas etapas:

  1. Upload Inicial (v1): Quando você cria um documento, o primeiro arquivo enviado é a versão base (v1). Esta versão é aprovada automaticamente.
  2. Novas Revisões (v2+): Ao fazer upload de uma nova planilha para o mesmo documento, ela entra como Pendente.
  3. Comparação Automática: O sistema analisa a nova versão comparando célula por célula com a última versão aprovada no banco de dados.
  4. Aprovação/Rejeição: Os auditores revisam as alterações e aprovam ou rejeitam a nova versão, registrando a decisão e comentários na trilha de auditoria.

Como a Comparação é Realizada

A comparação não requer o armazenamento dos arquivos físicos no servidor. O DocFlow extrai o conteúdo das planilhas para o formato estruturado JSON diretamente na memória e salva no banco de dados.

  • • Comparação por Célula: O sistema compara a linha R e coluna C da aba S em ambas as planilhas.
  • • Tratamento de Abas: Abas adicionadas ou removidas inteiras também são mapeadas e reportadas como modificações completas.
  • • Deduplicação de Versões: Arquivos com hash SHA-256 idêntico ao de uma versão anterior do mesmo documento são recusados no upload, evitando redundâncias.
